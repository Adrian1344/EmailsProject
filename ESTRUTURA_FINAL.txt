🚀 COMO USAR
────────────

DESENVOLVIMENTO LOCAL (3 passos)
1. Abra terminal na pasta do projeto
2. Execute: python dev_server.py 8000
3. Abra: http://localhost:8000

DEPLOY VERCEL (5 passos)
1. Crie repositório no GitHub (git push origin main)
2. Acesse Vercel e crie novo projeto
3. Selecione seu repositório
4. Upload arquivo alunos2.json
5. Pronto! Site está publicado

📊 API REFERENCE
────────────────

ENDPOINT:
  GET /api/search?query=termo

EXEMPLOS:
  curl "http://localhost:8000/api/search?query=BEATRIZ"
  
RESPOSTA (com resultados):
  {
    "success": true,
    "query": "BEATRIZ",
    "total": 1,
    "results": [
      {
        "nome": "BEATRIZ SOLARY...",
        "email": "beatriz.sgcosta@..."
      }
    ]
  }

RESPOSTA (sem resultados):
  {
    "success": true,
    "query": "XYZABC",
    "total": 0,
    "results": []
  }

🔐 COMO OS DADOS SÃO PROTEGIDOS
────────────────────────────────

1. ARQUIVO IGNORADO NO GIT
   .gitignore contém:
   - alunos2.json
   - alunos.json
   - .env
   
   ➜ Dados NUNCA aparecem no GitHub

2. API PROCESSA NO SERVIDOR
   Frontend faz:  GET /api/search?q=termo
   Backend busca: alunos2.json (servidor)
   Retorna:       apenas resultado
   
   ➜ Dados NUNCA são expostos no HTML

3. HEADERS DE SEGURANÇA
   Cache-Control: no-cache, no-store
   CORS: habilitado apenas para domínio autorizado
   
   ➜ Dados NUNCA são cacheados

4. UPLOAD SEGURO NO VERCEL
   Arquivo entra via Dashboard ou CLI
   Armazenado em servidor privado
   Acessível apenas pela API
   
   ➜ Dados PROTEGIDOS no servidor
